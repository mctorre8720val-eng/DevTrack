
Sidebar:
- Fixed
- Compact
- Clear active state
- Icon + label navigation

Main content:
- Max width container
- Organized sections
- Avoid unnecessary empty space

Mobile:
- Collapsible sidebar
- Bottom navigation optional
- Preserve readability

---

# Components

Create reusable components:

- Button
- Card
- Input
- Modal
- Navbar
- Sidebar
- Table
- Badge
- Empty State

Each component must have:
- Consistent spacing
- Hover states
- Loading states
- Error states

---

# Cards

Cards should:
- Group related information
- Use subtle borders
- Have clear purpose

Avoid:
- Every element becoming a card
- Large floating panels

---

# Colors

Primary:
Blue

Usage:
- Main actions
- Active navigation
- Links

Success:
Green

Usage:
- Completed tasks
- Success messages

Background:
Light neutral

Use:
- White surfaces
- Soft gray backgrounds

Do not introduce extra colors without reason.

---

# Typography

Use modern readable fonts.

Hierarchy:

Heading:
- Clear
- Medium weight

Body:
- Easy scanning

Labels:
- Smaller but readable

Avoid:
- Extremely large headings
- Thin unreadable text

---

# Spacing System

Use consistent spacing:

xs: 4px
sm: 8px
md: 16px
lg: 24px
xl: 32px
2xl: 48px

Do not use random spacing values.

---

# Responsive Behavior

Desktop first.

All components must adapt:

Desktop:
- Sidebar visible

Tablet:
- Reduced spacing

Mobile:
- Collapsible navigation
- Stacked layouts

---

# Implementation Rule

Before creating new UI:

1. Check existing components
2. Reuse existing patterns
3. Match the design system
4. Do not invent new styles unless necessary
